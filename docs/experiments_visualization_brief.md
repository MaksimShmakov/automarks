# Бриф на визуализацию экспериментов

## Зачем это нужно
Сейчас эксперименты хранятся и двигаются по статусам, но результат читается вручную из карточек. Нужен отдельный слой визуализации, чтобы быстро отвечать на три вопроса:
1. Что сейчас тестируется.
2. Какие гипотезы выигрывают или проигрывают.
3. Какие паттерны повторяются по ботам, веткам, типам изменений и метрикам.

## Цель MVP
Собрать экран или блок аналитики, где по всем экспериментам видно:
- в каком статусе находится гипотеза;
- на какую метрику она влияет;
- какой был сплит трафика;
- какие результаты у варианта A и B;
- чем закончился тест: успех, провал или ретест.

## Целевая аудитория
- Маркетолог: быстро видит, что тестируется и что масштабировать.
- Аналитик: получает структуру для разбора результатов и накопления знаний.
- Тимлид/руководитель: видит загрузку по экспериментам и качество гипотез.

## Что должно быть на экране
- Сводка по всем экспериментам за период.
- Отдельный блок по активным тестам.
- Отдельный блок по библиотеке завершенных гипотез.
- Возможность открыть карточку эксперимента и провалиться в детали.

## Основные виджеты
- Карточки-метрики:
  - всего экспериментов;
  - активных сейчас;
  - успешных;
  - провальных;
  - отправленных на ретест.
- График по динамике запусков:
  - сколько экспериментов запускали по неделям или месяцам.
- Воронка по статусам:
  - backlog -> draft -> в тесте -> на оценке -> успех/провал/ретест.
- Таблица завершенных экспериментов:
  - дата;
  - название;
  - бот;
  - ветка;
  - метрика;
  - сплит;
  - итоговый статус;
  - короткий вывод.
- Срез по типу гипотез:
  - стартовый экран;
  - сегментация;
  - номер;
  - подписка;
  - дожимы;
  - продажа;
  - свой вариант.

## Фильтры
- Период запуска и период завершения.
- Статус эксперимента.
- Бот.
- Ветка.
- Наличие A/B теста.
- Тип гипотезы.
- Метрика влияния.
- Ответственный.

## Что нужно уметь визуализировать
- Конверсию или целевую метрику до/после.
- Разницу между вариантом A и B.
- Частоту побед по типам гипотез.
- Частоту ретестов по ботам и веткам.
- Среднюю длительность теста.
- Долю гипотез, которые дошли до финального решения.

## Источники данных
База экспериментов уже содержит полезные поля:
- название эксперимента;
- статус;
- бот/ветка;
- признак A/B;
- варианты гипотез;
- метрика влияния;
- ожидаемое изменение;
- гипотеза;
- объем трафика;
- длительность;
- даты старта и окончания;
- dashboard_url;
- результаты по варианту A и B;
- комментарий;
- автор и даты обновления.

## Что желательно добавить в следующей итерации
- Отдельное поле "вывод" или "решение", чтобы не парсить свободный комментарий.
- Нормализованное поле "основная метрика" для единых фильтров.
- Числовые поля uplift / delta, если хотим строить сравнения без ручного текста.
- Теги по категории гипотезы и сегменту аудитории.

## Нефункциональные требования
- Быстро работает на списке из сотен экспериментов.
- Не требует ручной подготовки данных вне платформы.
- Имеет экспорт в таблицу для дальнейшего анализа.
- Понятен без отдельного онбординга для базового сценария.

## Критерии приемки MVP
- Можно за 30-60 секунд понять, какие тесты активны и чем закончились последние завершенные.
- Можно отфильтровать эксперименты по боту, ветке и периоду.
- Можно увидеть хотя бы один агрегированный график по итогам экспериментов.
- Можно открыть источник результата через dashboard_url.
- Можно использовать экран как базу знаний по завершенным гипотезам.
