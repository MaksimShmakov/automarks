# Automarks API

Документация актуальна для текущей версии Django-приложения.
Все URL относительны к базовому пути приложения.

## Аутентификация и форматы
- Формат запросов и ответов: JSON, если не указано иное.
- `GET /api/bot/<bot_name>/` открыт и не требует аутентификации.
- `POST /update_field/` доступен только авторизованным пользователям с ролью `admin`, `manager` или `marketer`.
- Даты принимаются в формате `YYYY-MM-DD`. Для месячных значений допускается `YYYY-MM`, тогда день автоматически считается как `01`.

Коды ответов:
- `200` — успешный ответ
- `400` — ошибка валидации или формата
- `401/403` — нет доступа
- `404` — объект не найден

## 1. Получение структуры бота

`GET /api/bot/<bot_name>/`

Возвращает данные о боте, его ветках и связанных UTM-тегах.

### Параметр пути
- `bot_name` — API-идентификатор бота из `marks.models.Bot.name`
- Для Telegram это username бота без `@`
- Для VK это числовой `group`

Маршрут не менялся: для VK используется тот же эндпоинт, просто с другим типом идентификатора.

### Успешный ответ
```json
{
  "bot": "python_course_bot",
  "branches": [
    {
      "name": "Главная ветка",
      "code": "ell01",
      "tags": [
        {
          "number": "ell010001",
          "utm_source": "ads",
          "utm_medium": "cpc",
          "utm_campaign": "ell01_campaign_1",
          "utm_term": "python",
          "utm_content": "banner",
          "url": "https://t.me/python_course_bot?start=ell010001"
        }
      ]
    }
  ]
}
```

### Формат ссылок в тегах
- Telegram: `https://t.me/<bot_name>?start=<tag_number>`
- VK: `https://vk.com/write-<group>?ref=<tag_number>`

### Ошибки
- `404 {"error": "Bot not found"}` или локализованный аналог — бот с указанным идентификатором отсутствует

## 2. Точечное обновление сущностей

`POST /update_field/`

Обновляет одно поле записи без полной формы.

### Тело запроса
```json
{
  "model": "plan",
  "id": 42,
  "field": "budget",
  "value": "150000.00"
}
```

### Параметры
- `model` — тип модели: `plan`, `report`, `tag`
- `id` — целочисленный идентификатор записи
- `field` — имя поля из разрешённого списка
- `value` — новое значение

### Успешный ответ
```json
{"success": true}
```
